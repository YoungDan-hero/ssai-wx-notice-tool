## 功能简介

SSAI-WX 通知小工具是一个 Windows 桌面端的微信群消息批量发送工具。

**它不会调用微信接口，也不会自动搜索群聊。** 工具的工作原理是：

1. 你手动在微信客户端里把目标群聊"拆分成独立窗口"（拖出主窗口）。
2. 工具通过 Win32 API 检测这些已拆分的微信窗口。
3. 你在工具的界面里编辑消息（文字、图片、视频、文档）。
4. 工具逐个激活微信窗口，通过系统剪贴板粘贴内容并模拟回车发送。

支持发送的内容类型：

- **文字** — 直接在输入框输入
- **图片** — PNG、JPG、GIF、BMP、TIFF、HEIC、WebP
- **视频** — MP4、MOV、M4V、AVI、MKV
- **文档** — `.txt` 和 `.docx`（`.doc` 格式需先另存为 `.docx`）

## 环境要求

| 项目 | 要求 |
|---|---|
| 操作系统 | Windows 10 / Windows 11 |
| Python | 3.11 或更新版本（源码运行时需要） |


## 源码运行（开发/调试）

如果你想直接运行源码而不打包，按以下步骤操作。

### 1. 克隆项目

```powershell
git clone git@github.com:eddylu93/ssai-wx-notice-tool.git
cd ssai-wx-notice-tool
```

### 2. 创建虚拟环境

```powershell
python -m venv .venv
```

### 3. 激活虚拟环境

```powershell
.venv\Scripts\Activate.ps1
```

> 如果 PowerShell 提示执行策略限制，先运行：
> ```powershell
> Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
> ```

### 4. 安装依赖

```powershell
pip install -r requirements.txt
```

### 5. 启动程序

```powershell
python app.py
```

程序启动后会显示一个仿微信聊天风格的窗口界面。

## 打包构建（生成 .exe）

打包后可以得到一个免安装的独立目录，包含 `.exe` 可执行文件，无需 Python 环境即可运行。

### 前置条件

- Windows 10 / 11
- Python 3.11 或更新版本（建议从 [python.org](https://www.python.org/downloads/) 安装）
- 已克隆项目并进入项目根目录

### 打包步骤

#### 方式一：使用打包脚本（推荐）

```powershell
cd ssai-wx-notice-tool

# 允许当前会话执行 PowerShell 脚本
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

# 运行打包脚本
packaging\windows\build_windows.ps1
```

脚本会自动完成以下操作：

1. 创建 `.venv` 虚拟环境
2. 安装所有依赖（包括 PyInstaller）
3. 使用 PyInstaller 打包为窗口应用（`--windowed`，无控制台黑窗口）
4. 输出到 `dist\SSAI-WX-Notice-Tool\`

打包完成后，可执行文件位于：

```
dist\SSAI-WX-Notice-Tool\SSAI-WX-Notice-Tool.exe
```

#### 方式二：手动打包

如果脚本执行有问题，也可以手动操作：

```powershell
# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
.venv\Scripts\Activate.ps1

# 安装依赖
pip install -r requirements.txt
pip install pyinstaller

# 打包
pyinstaller --noconfirm --clean --windowed --name "SSAI-WX-Notice-Tool" --icon assets\app_icon.ico --add-data "assets;assets" --hidden-import docx --hidden-import pyautogui --hidden-import win32clipboard --hidden-import win32con app.py
```

#### 打包产物说明

| 路径 | 说明 |
|---|---|
| `dist\SSAI-WX-Notice-Tool\` | 打包输出目录 |
| `dist\SSAI-WX-Notice-Tool\SSAI-WX-Notice-Tool.exe` | 主程序可执行文件 |
| `dist\SSAI-WX-Notice-Tool\_internal\` | PyInstaller 依赖文件（不要删除） |

> 整个 `dist\SSAI-WX-Notice-Tool\` 目录需要一起分发，不能只拷贝 `.exe` 文件。

#### 分发方式

- 直接将 `dist\SSAI-WX-Notice-Tool\` 文件夹压缩为 `.zip` 发送给使用者
- 使用者解压后双击 `SSAI-WX-Notice-Tool.exe` 即可运行
- 免安装，不需要 Python 环境

## 使用步骤

### 第一步：准备微信窗口

1. 打开并登录 Windows 微信客户端。
2. 找到本次要通知的群聊。
3. 将每个目标群聊**拆分成独立聊天窗口** — 方法是将群聊从微信主窗口中拖出来，使其成为独立窗口。
4. 确保拆分出的窗口**没有最小化**，且处于可见状态。

> 工具只会检测已拆分的独立窗口，不会读取微信主窗口里的群聊列表。

### 第二步：启动工具

- **打包版**：双击 `SSAI-WX-Notice-Tool.exe`
- **源码版**：在虚拟环境中运行 `python app.py`

### 第三步：刷新窗口列表

1. 程序启动后，点击界面上的 **"刷新窗口"** 按钮。
2. 右侧窗口列表会显示当前检测到的所有微信独立窗口。
3. 确认列表中**只包含本次要发送的目标窗口**。如果有多余窗口，请在微信中关闭或最小化它，然后再次点击"刷新窗口"。

### 第四步：编辑消息

在底部输入框中输入要发送的文字内容。

- **添加图片/视频**：点击输入框左侧的 `+` 按钮选择文件，也可以直接将图片/视频**拖拽**到输入框，或从剪贴板**粘贴**（Ctrl+V）。
- **添加文档**：点击 `+` 按钮选择 `.txt` 或 `.docx` 文件。
- 附件会在输入框下方显示预览，点击预览旁的 `×` 可删除。

### 第五步：发送

1. 点击 **"同步发送"** 按钮（或在输入框中按 Enter 键）。
2. 工具会先把消息显示为聊天气泡，然后逐个激活检测到的微信窗口，执行粘贴和发送。
3. 每个窗口之间会自动随机等待 1-4 秒，避免发送过快触发限制。
4. 发送过程中，右侧窗口列表会实时显示每个窗口的发送状态。

## 文档发送说明

工具支持发送 `.txt` 和 `.docx` 文档，处理方式如下：

### 文本拆分规则

- 文档内容按**段落**优先拆分。
- 单条消息超过 **300 字**时，会按标点符号和字数继续切分为多条消息。
- 上传文档后，界面会显示**文字条数**和**媒体文件数量**。

### .docx 媒体提取

- `.docx` 文件中嵌入的**图片和视频**会被自动提取出来。
- 每个媒体文件会作为**单独的发送任务**进入队列。
- 视频任务会额外等待更长时间，降低微信未上传完成就继续下一条的风险。

### 注意事项

- **文档模式和手动附件互斥** — 上传文档会清空当前的附件预览。
- `.doc` 格式不支持，请先用 Word 另存为 `.docx`。
- 每条拆分后的文字和每个媒体文件都会作为独立任务依次进入发送队列。

## 任务队列与发送控制

工具支持在发送过程中继续编辑和排队新消息：

### 排队发送

- 发送过程中，在输入框输入新消息并按 Enter 或点击按钮，新消息会进入**发送队列**。
- 当前消息处理完成后，工具会自动发送队列中的下一条。

### 暂停 / 继续

- 发送中点击 **"暂停"**，工具会在当前窗口动作结束后暂停队列，不会清空等待中的任务。
- 暂停后按钮变为 **"继续"**，点击后从暂停位置继续处理。

### 停止

- 发送中点击 **"停止"**，会清空等待队列。
- 已经开始粘贴或发送的当前动作无法撤回，工具会在当前窗口动作结束后停止处理后续窗口和后续任务。

### 窗口置顶

- 工具默认置顶显示，方便在微信旁边操作。
- 如果不想让它一直浮在最前面，可以取消 **"置顶"** 选项。

## 试运行模式

**强烈建议第一次使用时先进行试运行。**

1. 勾选界面上的 **"试运行，不真正发送"** 复选框。
2. 点击"同步发送"。
3. 工具会按检测到的窗口列表走完整个流程并记录结果，但**不会真正粘贴或发送任何内容**。
4. 检查发送记录，确认窗口检测和流程无误后，取消勾选再正式发送。

## 发送记录与日志

### 日志文件位置

Windows 下的日志文件路径：

```
%APPDATA%\SSAI-WX 通知小工具\send_log.jsonl
```

通常展开为：

```
C:\Users\<你的用户名>\AppData\Roaming\SSAI-WX 通知小工具\send_log.jsonl
```

### 日志格式

每行是一条 JSON 记录，包含以下字段：

| 字段 | 说明 |
|---|---|
| 时间 | 发送时间戳 |
| 窗口标题 | 目标微信窗口的标题 |
| 文字摘要 | 发送内容的摘要 |
| 附件文件名 | 附件文件名（如有） |
| 状态 | 发送结果（成功/失败） |
| 错误信息 | 失败时的错误详情 |

### 快速打开日志目录

按 `Win + R`，输入以下路径后回车：

```
%APPDATA%\SSAI-WX 通知小工具
```

## 常见问题

### 没有检测到微信窗口

- 确认微信已经打开并**已登录**。
- 确认群聊已经**拆分成独立窗口**（从主窗口拖出来），而不是在微信主窗口的聊天列表里。
- 确认拆分出的窗口**没有最小化**。
- 点击 **"刷新窗口"** 重新检测。
- 检查 Windows 任务管理器中微信进程名是否为 `WeChat.exe` 或 `Weixin.exe`。

### 图片或视频没有发送成功

- 确认已安装所有依赖（`pip install -r requirements.txt`）。
- 图片建议使用 PNG 或 JPG 格式，部分冷门格式可能不被微信支持。
- 视频文件过大会导致微信上传时间较长，工具会自动等待，但如果等待不够可能导致后续消息跳过。
- 发送过程中不要手动操作剪贴板（如复制其他内容），否则会干扰工具的粘贴操作。

### 工具粘贴了内容但没有实际发送

- 确认微信窗口的**输入框处于可用状态**（没有被其他弹窗遮挡）。
- 确认微信窗口**没有被最小化**。
- 检查发送日志中的错误信息。

### 粘贴到了错误的窗口

- 发送前确认右侧窗口列表中**只包含目标群聊窗口**。
- 关闭或最小化不需要发送的微信窗口，然后点击"刷新窗口"重新检测。
- 发送过程中不要手动切换窗口焦点。

### .doc 文件无法上传

- 工具只支持 `.docx` 格式，不支持旧版 `.doc` 格式。
- 用 Word 打开 `.doc` 文件，选择"另存为" → `.docx` 格式，然后重新上传。

### Windows Defender / 杀毒软件拦截

- PyInstaller 打包的 `.exe` 可能被 Windows Defender 误报。
- 如果遇到拦截，可以将 `SSAI-WX-Notice-Tool.exe` 所在目录添加到 Windows Defender 的排除列表中：
  1. 打开"Windows 安全中心"。
  2. 进入"病毒和威胁防护" → "管理设置"。
  3. 在"排除项"中添加 `dist\SSAI-WX-Notice-Tool\` 目录。

### 程序启动后没有反应

- 确认使用的是 Windows 10 或 11。
- 如果是打包版，尝试从命令行启动 `.exe` 查看错误输出：
  ```powershell
  cd dist\SSAI-WX-Notice-Tool
  .\SSAI-WX-Notice-Tool.exe
  ```
- 如果是源码版，确认虚拟环境已激活且依赖已安装。

### 发送速度太快/太慢

- 工具在每个窗口之间会自动随机等待 **1-4 秒**（1400-2200ms），这是内置的固定间隔，无法手动调整。
- 如果需要发送大量窗口，建议分批操作，每批 10-15 个窗口。

### 会不会误发

- 工具**不会自动搜索群聊**，也不会在后台静默发送。
- 发送前会显示检测到的窗口列表，你有充分的时间确认。
- 不确定时，先使用**试运行模式**走一遍流程。
